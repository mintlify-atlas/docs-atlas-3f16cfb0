## Overview

The Resource Management module controls your production capacity by tracking machines, workstations, and human operators. It serves as the foundation for work order assignments, capacity planning, and utilization tracking.

## Resource Types

<Tabs>
  <Tab title="HUMANO (Human)">
    **Operator or Worker Position**
    
    Represents a person or role in production:
    - Individual operators (e.g., "Juan Pérez")
    - Generic roles (e.g., "Soldador Turno A")
    - Workstation positions (e.g., "Operador Torno 1")
    
    **Usage:**
    - Matched with user accounts for assignment
    - Name should match user's `fullName` for auto-linking
    - Used in operator terminal assignment logic
    - Tracked in productivity reports

    ```

    Type: HUMANO
    Sector: Mecanizado
    Status: DISPONIBLE
    ```

  <Tab title="MAQUINA (Machine)">

    - CNC machines (e.g., "Torno CNC Haas ST-10")

    - Workbenches (e.g., "Banco de Ensamble 3")

    **Usage:**
    - Track machine availability
    - Schedule maintenance windows

    - Link to specific work orders

## Resource Statuses

Each resource has a current operational status:

### DISPONIBLE (Available)

- **Meaning**: Resource is ready for assignment
- **Color**: Green badge
- **Use for**: Idle equipment or available operators
- **Actions**: Can be assigned to new work orders

### OCUPADO (Busy)

- **Meaning**: Currently assigned to active work order
- **Color**: Blue badge  
- **Use for**: Resources in use on EN_PROCESO orders
- **Actions**: Cannot assign to additional orders (capacity full)

### MANTENIMIENTO (Maintenance)

- **Meaning**: Under repair or scheduled maintenance
- **Color**: Yellow/warning badge
- **Use for**: Planned downtime, repairs, calibration
- **Actions**: Should not be assigned to orders
- **Notes**: Document maintenance schedule in notes field

### FUERA_DE_LINEA (Offline)

- **Meaning**: Permanently unavailable or decommissioned
- **Color**: Red/destructive badge
- **Use for**: Broken equipment, terminated employees
- **Actions**: Excluded from assignment dropdowns
- **Notes**: Use instead of deleting (preserves history)

## User Workflows

<Steps>
  <Step title="Create New Resource">
    1. Click **+ Nueva Estación** in page header
    2. Select resource type:
       - **HUMANO (Persona / Puesto)**: For operators
       - **MAQUINA (Física / Virtual)**: For equipment
    3. Enter resource name (e.g., "Torno CNC 1" or "Pérez, Juan")
    4. Set sector/area (e.g., "Mecanizado Ligero")
    5. Add optional notes (maintenance schedule, capabilities)
    6. Click **Asignar Alta al Taller**
    7. Resource appears in table with DISPONIBLE status
  </Step>
  
  <Step title="Update Resource Status">
    1. Find resource in main table
    2. In "Disponibilidad" column, click status dropdown
    3. Select new status:
       - DISPONIBLE (ready for work)
       - OCUPADO (currently assigned)
       - MANTENIMIENTO (under repair)
       - FUERA_DE_LINEA (offline)
    4. Status updates immediately
    5. Change reflected in assignment dropdowns
  </Step>
  
  <Step title="Assign Resource to Work Order">
    1. Navigate to Work Orders page
    2. Click **⚙️ Gest.** on desired work order
    3. In "Asignar Puesto" section:
       - For HUMANO: Select operator from dropdown
       - System matches operator name with user account
       - Automatically links to matching HUMANO resource
    4. Click **Asignar** button
    5. Assignment creates:
       - Link between WO and resource
       - Link between WO and user
       - Visibility in operator's terminal
  </Step>
  
  <Step title="Search and Filter Resources">
    1. Use search bar: "🔍 Buscar recurso o sector..."
    2. Type resource name or sector
    3. Or use "Todos los Tipos" dropdown:
       - MAQUINA: Show only equipment
       - HUMANO: Show only operators
    4. Table filters in real-time
    5. Count badge shows filtered results
  </Step>
</Steps>

## Assignment Logic

### Operator Matching

When assigning work orders, the system matches operators to resources:

```typescript
// Matching algorithm (WorkOrdersPage.tsx:74-79)
const operatorOptions = useMemo(() => {
  const res = resourcesQuery.data ?? [];
  return (operatorsQuery.data ?? []).map(o => {
    const r = res.find(x => 
      x.name.trim().toLowerCase() === o.fullName.trim().toLowerCase()
    );
    return r ? { userId: o.id, fullName: o.fullName, resourceId: r.id } : null;
  }).filter(Boolean);
}, [operatorsQuery.data, resourcesQuery.data]);
```

**Requirements:**
1. User account exists with role OPERARIO
2. User account is active (`isActive: true`)
3. HUMANO resource exists with matching name
4. Names match exactly (case-insensitive, trimmed)

**Example:**
```
User Account:
  fullName: "Rodriguez, Carlos"
  role: OPERARIO
  isActive: true

Resource:
  type: HUMANO
  name: "Rodriguez, Carlos"
  
✅ Matches - appears in assignment dropdown
```

### Assignment Creation

When you assign an operator to a work order:

```bash
POST /api/work-orders/{id}/assignments
{
  "resourceId": "resource-uuid",
  "userId": "user-uuid"
}
```

System creates:
- Assignment record linking WO + Resource + User
- Work order becomes visible in operator's terminal
- Status may auto-update based on rules
- Assignment tracked in reports

## Best Practices

<AccordionGroup>
  <Accordion title="Consistent Naming Convention">
    Establish and document naming standards:
    
    **For HUMANO resources:**
    - Match user account `fullName` exactly
    - Format: "Lastname, Firstname" or "Firstname Lastname"
    - Example: "Pérez, Carlos" or "Carlos Pérez"
    
    **For MAQUINA resources:**
    - Include type, brand, model if relevant
    - Number duplicates sequentially
    - Example: "Torno CNC Haas ST-10 #1", "Torno CNC Haas ST-10 #2"
  </Accordion>
  
  <Accordion title="Active Status Management">
    Keep status fields current:
    - Update to MANTENIMIENTO before planned maintenance
    - Set to FUERA_DE_LINEA instead of deleting
    - Use notes field to explain status changes
    - Review status weekly for accuracy
  </Accordion>
  
  <Accordion title="Sector Organization">
    Use sectors for meaningful grouping:
    - Align with physical shop floor layout
    - Or organize by skill/capability
    - Document sector definitions
    - Train staff on which sector to use
  </Accordion>
  
  <Accordion title="Notes Field Usage">
    Document important information:
    
    **For machines:**
    - Maintenance schedule ("PM cada 500 hrs")
    - Repair history ("Husillo reemplazado Feb 2026")
    - Capabilities ("Max RPM 3000, Chuck 12in")
    - Certifications or calibrations
    
    **For operators:**
    - Certifications ("Cert. Soldadura MIG/TIG")
    - Specializations ("Experto en piezas complejas")
    - Shift assignments ("Turno A: 06:00-14:00")
    - Training status
  </Accordion>

**Database Schema (Prisma):**

```prisma
model Resource {
  id        String         @id @default(uuid())
  type      ResourceType   // HUMANO | MAQUINA
  name      String         @db.VarChar(120)
  sector    String?        @db.VarChar(120)
  status    ResourceStatus @default(DISPONIBLE)
  notes     String?        @db.VarChar(500)
  
  assignments Assignment[]
  
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  
  @@index([type])
  @@index([status])
}

enum ResourceType {
  HUMANO
  MAQUINA
}

enum ResourceStatus {
  DISPONIBLE
  OCUPADO
  MANTENIMIENTO
  FUERA_DE_LINEA
}
```
