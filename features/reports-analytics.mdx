    ```typescript
      };
    ```

  <Tab title="Financial Summary">
    **Estimated Financial Metrics** (Period: 3M, 6M, or 12M)
    
    | Metric | Calculation | Color |
    |--------|-------------|-------|
    | Facturación estimada | Sum of `estimatedCost` for closed orders | Default |
    | Ganancia estimada | Sum where real cost < estimated cost | Green |
    | Pérdida estimada | Sum where real cost > estimated cost | Red |
    | Costo real materiales | Sum of all material consumptions | Default |
    | Margen neto estimado | Ganancia - Pérdida | Green/Red |
    
    **Formula:**
    ```typescript
    for each closed WorkOrder {
      estimatedCost = wo.estimatedCost;
      realCost = sum(wo.materialConsumptions.map(
        c => c.quantity * c.unitCostSnapshot
      ));
      
      if (realCost < estimatedCost) {
        gain += (estimatedCost - realCost);
      } else {
        loss += (realCost - estimatedCost);
      }
    }
    
    margin = gain - loss;
    ```
    
    <Note>
      Financial metrics are **estimates** based on quotation values and material costs only. Labor, overhead, and other expenses are not included in the current version.
    </Note>
  </Tab>

### Operational Alerts

**Alertas operativas** panel shows deviation warnings:

#### Alert Criteria

**Time Deviation:**
```typescript
const timeDeviationPct = (
  (workedHours - estimatedHours) / estimatedHours
) * 100;

if (timeDeviationPct > criticalTimeThreshold) {
  severity = 'critical';  // 🔴 Red
} else if (timeDeviationPct > warningTimeThreshold) {
  severity = 'warning';   // 🟡 Yellow
}
```

**Cost Deviation:**
```typescript
const costDeviationPct = (
  (realMaterialCost - estimatedCost) / estimatedCost
) * 100;

if (costDeviationPct > criticalCostThreshold) {
  severity = 'critical';
} else if (costDeviationPct > warningCostThreshold) {
  severity = 'warning';
}
```

**Default Thresholds (Configurable):**
- Warning Time: +20%
- Critical Time: +40%
- Warning Cost: +15%
- Critical Cost: +30%

#### Alert Display

**Header Badge:**
- 🔴 {N} crítica(s) - Red
- 🟡 {N} aviso(s) - Yellow

**Alert List Item:**
```
[🔴/🟡] OT-2026-045 · Industria Aceros SA
         Tiempo excedido: +45% · Costo extra: +18%
         [CRÍTICO/AVISO]
```

**No Alerts State:**
```
✅ Todas las órdenes dentro de los umbrales normales.
```

## Using Recharts Visualizations

All charts use the Recharts library:

### Line Chart Example

```tsx
import { LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip, Legend } from 'recharts';

<ResponsiveContainer width="100%" height={220}>
  <LineChart data={lineData}>
    <CartesianGrid strokeDasharray="3 3" stroke="var(--border)" />
    <XAxis dataKey="mes" tick={{ fontSize: 11, fill: 'var(--ink-soft)' }} />
    <YAxis tick={{ fontSize: 11, fill: 'var(--ink-soft)' }} />
    <Tooltip content={<CustomTooltip />} />
    <Legend wrapperStyle={{ fontSize: '0.8rem' }} />
    <Line type="monotone" dataKey="OTs cerradas" stroke="#fb923c" strokeWidth={2} />
    <Line type="monotone" dataKey="Horas prom./OT" stroke="#38bdf8" strokeWidth={2} />
  </LineChart>
</ResponsiveContainer>
```

### Custom Tooltip

```tsx
const CustomTooltip = ({ active, payload, label }) => {
  if (!active || !payload?.length) return null;
  return (
    <div className="custom-tooltip">
      <p className="label">{label}</p>
      {payload.map((entry, i) => (
        <p key={i} style={{ color: entry.color }}>
          {entry.name}: <strong>{entry.value}</strong>
        </p>
      ))}
    </div>
  );
};
```

### Pie Chart Example

```tsx
import { PieChart, Pie, Cell, Tooltip } from 'recharts';

const COLORS = ['#38bdf8', '#fb923c', '#a78bfa', '#34d399', '#f87171'];

<PieChart>
  <Pie
    data={pieData}
    cx="50%"
    cy="50%"
    innerRadius={55}
    outerRadius={85}
    paddingAngle={3}
    dataKey="value"
    label={({ name, percent }) => `${name} ${(percent * 100).toFixed(0)}%`}
  >
    {pieData.map((entry, index) => (
      <Cell key={`cell-${index}`} fill={COLORS[index % COLORS.length]} />
    ))}
  </Pie>
  <Tooltip />
</PieChart>
```

## Best Practices

<AccordionGroup>
  <Accordion title="Regular Review Cadence">
    Establish review schedules:
    
    **Daily (Supervisor):**
    - Check delayed orders
    - Review active alerts
    - Monitor operator workload
    
    **Weekly (Management):**
    - Analyze deviation trends
    - Identify recurring issues
    - Adjust resource allocation
    
    **Monthly (Executive):**
    - Review financial margins
    - Compare periods (3M vs 6M vs 12M)
    - Set improvement goals
  </Accordion>
  
  <Accordion title="Threshold Tuning">
    Adjust alert thresholds based on your operation:
    
    **Tight tolerances** (high precision work):
    - Warning: +10% time, +10% cost
    - Critical: +20% time, +15% cost
    
    **Standard tolerances** (default):
    - Warning: +20% time, +15% cost
    - Critical: +40% time, +30% cost
    
    **Loose tolerances** (prototype/R&D):
    - Warning: +50% time, +30% cost
    - Critical: +100% time, +50% cost
    
    Configure in: Resources > Company Settings
  </Accordion>
  
  <Accordion title="Data Quality">
    Ensure accurate reporting:
    
    - Train operators on timely event logging
    - Verify material consumption records daily
    - Audit outlier deviations for data errors
    - Correct mistakes promptly (affects future reports)
    - Document any manual adjustments
  </Accordion>
  
  <Accordion title="Action on Insights">
    Use reports to drive improvement:
    
    **High time deviation:**
    - Review work complexity
    - Provide additional training
    - Update estimates for future
    - Check equipment issues
    
    **High cost deviation:**
    - Verify material pricing
    - Check for waste/scrap
    - Review process efficiency
    - Renegotiate supplier contracts
    
    **Frequent delays:**
    - Improve planning accuracy
    - Add buffer to commitments
    - Identify bottlenecks
    - Increase capacity in constraints
  </Accordion>
</AccordionGroup>

```typescript

    },
    include: {
      client: true,
```
