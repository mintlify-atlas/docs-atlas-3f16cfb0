## Base URL

All API requests should be made to:

```
http://your-domain.com/api
```

The API uses the `/api` prefix for all endpoints.

## Authentication

The DRAIT Mini-MES API uses **JWT (JSON Web Token)** authentication. Most endpoints require a valid JWT token to be included in the request headers.

To authenticate:

1. Obtain a JWT token by calling the `/api/auth/login` endpoint with valid credentials
2. Include the token in subsequent requests using the `Authorization` header:

```bash
Authorization: Bearer <your-jwt-token>
```

See the [Authentication](/api/authentication) page for detailed information.

## Response Format

All API responses follow standard HTTP status codes and return JSON data.

### Success Responses

**200 OK** - Request succeeded
```json
{
  "id": "clx123abc",
  "name": "Example Resource",
  "createdAt": "2026-03-13T10:00:00.000Z"
}
```

**201 Created** - Resource created successfully
```json
{
  "id": "clx123abc",
  "message": "Resource created successfully"
}
```

### Error Responses

**400 Bad Request** - Invalid request parameters
```json
{
  "statusCode": 400,
  "message": ["name should not be empty", "email must be an email"],
  "error": "Bad Request"
}
```

**401 Unauthorized** - Missing or invalid authentication
```json
{
  "statusCode": 401,
  "message": "Unauthorized"
}
```

**403 Forbidden** - Insufficient permissions
```json
{
  "statusCode": 403,
  "message": "Forbidden resource"
}
```

**404 Not Found** - Resource not found
```json
{
  "statusCode": 404,
  "message": "Resource not found"
}
```

**500 Internal Server Error** - Server error
```json
{
  "statusCode": 500,
  "message": "Internal server error"
}
```

## Rate Limiting

The API implements rate limiting using **@nestjs/throttler** to protect against abuse.

### Default Limits

Most endpoints are subject to standard rate limiting.

### Login Endpoint

The `/api/auth/login` endpoint has strict rate limiting:
- **5 attempts per minute** per IP address
- Exceeding this limit will result in a **429 Too Many Requests** response

```json
{
  "statusCode": 429,
  "message": "ThrottlerException: Too Many Requests"
}
```

## Multi-Tenancy

The API is designed with **multi-tenancy** support. Each company (tenant) has isolated data:

- Users belong to a specific company via `companyId`
- All resources (clients, quotations, work orders, etc.) are scoped to a company
- JWT tokens include the `companyId` claim to ensure data isolation
- Cross-company data access is prevented at the application level

## Common Patterns

### Pagination

Some list endpoints support pagination via query parameters:

<ParamField query="page" type="number" default="1">
  Page number (1-indexed)
</ParamField>

<ParamField query="limit" type="number" default="50">
  Number of items per page
</ParamField>

### Filtering

Many endpoints support filtering via query parameters:

<ParamField query="status" type="string">
  Filter by status
</ParamField>

<ParamField query="search" type="string">
  Search by name or description
</ParamField>

### Soft Deletes

Most resources use **soft deletes** with an `isActive` flag:
- Deleted resources are marked as `isActive: false`
- They remain in the database for audit purposes
- List endpoints typically exclude inactive resources by default

## Timestamps

All resources include standard timestamp fields:

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the resource was created
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp when the resource was last updated
</ResponseField>

## ID Format

All resource IDs use **CUID** format (Collision-resistant Unique Identifier):
- Example: `clx123abc456def789`
- Automatically generated by Prisma
- URL-safe and sortable

## Validation

The API uses **class-validator** for request validation:
- Invalid requests return **400 Bad Request** with detailed error messages
- Required fields must be provided
- Field types and formats are strictly enforced
- String lengths and numeric ranges are validated
