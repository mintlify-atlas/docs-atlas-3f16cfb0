## GET /api/work-orders

Retrieve all work orders with filtering options.

### Query Parameters

<ParamField query="status" type="string">
  Filter by status: PENDIENTE, PLANIFICADA, EN_PROCESO, PAUSADA, FINALIZADA, ENTREGADA, CANCELADA, RETRABAJO
</ParamField>

<ParamField query="search" type="string">
  Search by code, title, or description
</ParamField>

<ParamField query="assignedToMe" type="boolean">
  Filter work orders assigned to current user (use `true`)
</ParamField>

### Response

Returns an array of work order objects.

<ResponseField name="id" type="string">Work order ID</ResponseField>
<ResponseField name="companyId" type="string">Company ID</ResponseField>
<ResponseField name="clientId" type="string">Client ID</ResponseField>
<ResponseField name="quotationId" type="string">Related quotation ID</ResponseField>
<ResponseField name="code" type="string">Work order code (e.g., OT-2026-001)</ResponseField>
<ResponseField name="title" type="string">Work order title</ResponseField>
<ResponseField name="description" type="string">Detailed description</ResponseField>
<ResponseField name="status" type="string">Current status</ResponseField>
<ResponseField name="priority" type="number">Priority (1=high, 5=low)</ResponseField>
<ResponseField name="plannedDate" type="string">Planned start date</ResponseField>
<ResponseField name="commitmentDate" type="string">Commitment/deadline date</ResponseField>
<ResponseField name="startedAt" type="string">Actual start timestamp</ResponseField>
<ResponseField name="finishedAt" type="string">Finish timestamp</ResponseField>
<ResponseField name="deliveredAt" type="string">Delivery timestamp</ResponseField>
<ResponseField name="estimatedTimeMin" type="number">Estimated time in minutes</ResponseField>
<ResponseField name="estimatedCost" type="number">Estimated cost</ResponseField>
<ResponseField name="notes" type="string">Internal notes</ResponseField>
<ResponseField name="attachmentUrl" type="string">Delivery receipt URL</ResponseField>
<ResponseField name="isSigned" type="boolean">Receipt signed status</ResponseField>
<ResponseField name="dashboardUrl" type="string">Monitoring dashboard URL</ResponseField>
<ResponseField name="createdAt" type="string">Creation timestamp</ResponseField>
<ResponseField name="updatedAt" type="string">Last update timestamp</ResponseField>

### Example

<CodeGroup>
```bash cURL (All work orders)
curl "http://localhost:3000/api/work-orders" \
  -H "Authorization: Bearer <token>"
```

```bash cURL (Assigned to me)
curl "http://localhost:3000/api/work-orders?assignedToMe=true" \
  -H "Authorization: Bearer <token>"
```

```bash cURL (Filter by status)
curl "http://localhost:3000/api/work-orders?status=EN_PROCESO" \
  -H "Authorization: Bearer <token>"
```
</CodeGroup>

## POST /api/work-orders

Create a new work order manually.

<Warning>
  **Required Roles:** DUENO, SUPERVISOR, or ADMIN
</Warning>

### Request Body

<ParamField body="code" type="string" required>
  Unique work order code (e.g., OT-2026-001)
</ParamField>

<ParamField body="title" type="string" required>
  Work order title
</ParamField>

<ParamField body="description" type="string" required>
  Detailed description
</ParamField>

<ParamField body="clientId" type="string" required>
  Client ID
</ParamField>

<ParamField body="quotationId" type="string">
  Related quotation ID (optional)
</ParamField>

<ParamField body="priority" type="number" required>
  Priority level (1-5, where 1=high, 5=low)
</ParamField>

<ParamField body="plannedDate" type="string">
  Planned start date (ISO 8601)
</ParamField>

<ParamField body="commitmentDate" type="string">
  Commitment/deadline date (ISO 8601)
</ParamField>

<ParamField body="estimatedTimeMin" type="number" required>
  Estimated time in minutes (min: 0)
</ParamField>

<ParamField body="estimatedCost" type="number" required>
  Estimated cost (min: 0)
</ParamField>

<ParamField body="notes" type="string">
  Internal notes
</ParamField>

<ParamField body="dashboardUrl" type="string">
  Monitoring dashboard URL
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/api/work-orders \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "code": "OT-2026-010",
    "title": "Emergency Repair Order",
    "description": "Urgent repair for client equipment",
    "clientId": "clx789ghi",
    "priority": 1,
    "commitmentDate": "2026-03-14T17:00:00.000Z",
    "estimatedTimeMin": 480,
    "estimatedCost": 5000.00,
    "notes": "High priority - coordinate with client"
  }'
```
</CodeGroup>

## PATCH /api/work-orders/:id/status

Change work order status with automatic timestamp management.

<Warning>
  **Required Roles:** DUENO, SUPERVISOR, ADMIN, or OPERARIO
</Warning>

### Path Parameters

<ParamField path="id" type="string" required>
  Work order ID
</ParamField>

### Request Body

<ParamField body="status" type="string" required>
  New status: PENDIENTE, PLANIFICADA, EN_PROCESO, PAUSADA, FINALIZADA, ENTREGADA, CANCELADA, RETRABAJO
</ParamField>

<ParamField body="note" type="string">
  Optional note about status change
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X PATCH http://localhost:3000/api/work-orders/clxw123abc/status \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"status": "EN_PROCESO"}'
```
</CodeGroup>

<Note>
  Status transitions automatically update `startedAt`, `finishedAt`, and other timestamps based on workflow logic.
</Note>

## POST /api/work-orders/:id/events

Add an operation log event to track work order progress.

<Warning>
  **Required Roles:** DUENO, SUPERVISOR, ADMIN, or OPERARIO
</Warning>

### Path Parameters

<ParamField path="id" type="string" required>
  Work order ID
</ParamField>

### Request Body

<ParamField body="eventType" type="string" required>
  Event type: OT_CREADA, ASIGNADO, INICIO, PAUSA, REANUDACION, FINALIZACION, CAMBIO_ESTADO, MATERIAL_CONSUMIDO, NOTA
</ParamField>

<ParamField body="resourceId" type="string">
  Related resource ID
</ParamField>

<ParamField body="pauseReason" type="string">
  Reason for pause (required if eventType is PAUSA)
</ParamField>

<ParamField body="note" type="string">
  Additional notes
</ParamField>

### Example

<CodeGroup>
```bash cURL
curl -X POST http://localhost:3000/api/work-orders/clxw123abc/events \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "eventType": "PAUSA",
    "pauseReason": "Machine maintenance required",
    "note": "Will resume after maintenance"
  }'
```
</CodeGroup>
