## Prerequisites

Before installing DRAIT Mini-MES, ensure you have the following installed on your system:

- **Docker** (version 20.10 or higher)
- **Docker Compose** (version 2.0 or higher)
- At least **2GB of available RAM**
- At least **5GB of available disk space**

<Note>
DRAIT Mini-MES is designed for on-premise deployment and runs entirely in Docker containers.
</Note>

## System Requirements

### Minimum Requirements

- **CPU**: 2 cores
- **RAM**: 2GB
- **Storage**: 5GB
- **OS**: Linux, macOS, or Windows with WSL2

### Recommended Requirements

- **CPU**: 4 cores or more
- **RAM**: 4GB or more
- **Storage**: 10GB or more
- **OS**: Linux (Ubuntu 20.04+ or similar)

## Installation Steps

<Steps>
  <Step title="Clone the repository">
    Clone the DRAIT Mini-MES repository to your local machine:
    
    <CodeGroup>
    ```bash bash
    git clone <repository-url>
    cd Drait-mini-MES
    ```
    
    ```powershell PowerShell
    git clone <repository-url>
    cd Drait-mini-MES
    ```
    </CodeGroup>
  </Step>

  <Step title="Create environment configuration">
    Copy the example environment file to create your configuration:
    
    <CodeGroup>
    ```bash bash
    cp .env.example .env
    ```
    
    ```powershell PowerShell
    Copy-Item .env.example .env
    ```
    </CodeGroup>
    
    <Note>
    The default configuration works for local development. For production deployment, see the [Configuration](/getting-started/configuration) guide.
    </Note>
  </Step>

  <Step title="Build and start the containers">
    Use Docker Compose to build and start all services:
    
    <CodeGroup>
    ```bash bash
    docker compose up --build -d
    ```
    
    ```powershell PowerShell
    docker compose up --build -d
    ```
    </CodeGroup>
    
    This command will:
    - Build the backend and frontend Docker images
    - Pull the PostgreSQL 16 and Nginx images
    - Start all four containers (database, backend, frontend, nginx)
    - Run in detached mode (`-d` flag)
    
    <Warning>
    The initial build may take 5-10 minutes depending on your internet connection and system performance.
    </Warning>
  </Step>

  <Step title="Initialize the database">
    Apply the database schema and seed demo data:
    
    <CodeGroup>
    ```bash bash
    docker compose exec drait-backend npm run prisma:migrate
    docker compose exec drait-backend npm run prisma:seed
    ```
    
    ```powershell PowerShell
    docker compose exec drait-backend npm run prisma:migrate
    docker compose exec drait-backend npm run prisma:seed
    ```
    </CodeGroup>
    
    <Note>
    The seed command creates demo users, clients, materials, resources, and sample work orders for testing.
    </Note>
  </Step>
</Steps>

## Verify Installation

After installation, verify that all services are running correctly:

<Steps>
  <Step title="Check container status">
    <CodeGroup>
    ```bash bash
    docker compose ps
    ```
    
    ```powershell PowerShell
    docker compose ps
    ```
    </CodeGroup>
    
    All four containers should show status `Up` or `running`:
    - `drait-db`
    - `drait-backend`
    - `drait-frontend`
    - `drait-nginx`
  </Step>

  <Step title="Test the API health endpoint">
    Open your browser or use curl to check the API:
    
    ```bash
    curl http://localhost/api/health
    ```
    
    You should receive a successful response from the backend.
  </Step>

  <Step title="Access the web interface">
    Open your browser and navigate to:
    
    ```
    http://localhost/
    ```
    
    You should see the DRAIT Mini-MES login page.
  </Step>
</Steps>

<Note>
If you encounter any issues, check the container logs:

```bash
docker compose logs drait-backend
docker compose logs drait-frontend
docker compose logs drait-db
```
</Note>

## Troubleshooting

### Port Conflicts

If port 80 or 5432 is already in use, edit your `.env` file:

```bash
HTTP_PORT=8080        # Change from 80
POSTGRES_PORT=5433    # Change from 5432
```

Then restart the containers:

```bash
docker compose down
docker compose up -d
```

### Database Connection Issues

If the backend cannot connect to the database, ensure:

1. The database container is healthy: `docker compose ps drait-db`
2. The `DATABASE_URL` in `.env` matches your database credentials
3. Wait for the database health check to pass (may take 30-60 seconds on first start)

### Container Build Failures

If Docker build fails:

1. Ensure you have enough disk space: `docker system df`
2. Clean up old images: `docker system prune -a`
3. Try building again with `--no-cache`: `docker compose build --no-cache`
